OpenIdAjaxTextBox specification

Requirements

  1. User supplied text value preserved across postbacks
  2. Successful authentication results, if any, preserved across postbacks both in the user agent and server contexts.
  3. Asserting Provider's logo, where available, should appear in the place of the OpenID logo when successful assertion is provided.
  4. Identity assertion should be verified by the RP web server during the first postback of that web page where a positive assertion is available.
  5. Visual indication of when discovery and authentication are taking place.
  6. Authentication to be attempted in asynchronous immediate mode with each Provider authorized by the claimed identifier until an authentication is achieved if possible.
  7. If no OpenID endpoint will assert the user's identity in immediate mode, assist the user in "setup" mode authentication with the preferred Provider using a popup window.
  8. In a single page visit, all user supplied identifiers entered into the box resulting in discovery will have the discovery results saved in temporary memory  at the user agent, and will be reused rather than performing rediscovery if the user re-enters a previously entered identifier.
  9. In a single page visit, if the user enters multiple user supplied identifiers that on discovery resolve to the same claimed identifier, any authentication attempt made on that claimed identifier will be used for the other user supplied identifiers that resolve to the same claimed identifier.  For example, if the user types "blog.nerdbank.net" and authenticates to it, then changes the value to "http://blog.nerdbank.net" then when discovery reveals this as the same identifier, the successful authentication from "blog.nerdbank.net" will be applied to the new identifier implicitly.

Non-requirements

  1. Multiple authentication attempt logs preserved across postbacks

Design considerations

A throttle should be placed to limit how many immediate requests are made to Providers simultaneously, to avoid XRDS files with many Providers causing the login control to initiate so many connections that the browser slows down.
